\chapter{Présentation générale}\label{pruxe9sentation-guxe9nuxe9rale}

\textbf{AI Test Plan Generator} est une application web destinée à
assister les équipes qualité, validation et vérification dans la
création de plans de test à partir de documents techniques.

L'application permet d'importer des documents de spécification, d'en
extraire des exigences, de générer des plans de test et des cas de test,
puis de vérifier la couverture entre les exigences et les tests générés.

Le projet ne se limite pas à une simple interface de discussion avec un
modèle d'intelligence artificielle. Il s'agit d'une application complète
comprenant :

\begin{itemize}
\tightlist
\item
  une interface web ;
\item
  une API backend ;
\item
  une base de données ;
\item
  un système d'authentification ;
\item
  un pipeline IA multi-agents ;
\item
  une gestion documentaire ;
\item
  une traçabilité entre documents, exigences et tests ;
\item
  un chatbot contextuel ;
\item
  un suivi des ressources, du planning et des coûts LLM.
\end{itemize}

\chapter{Objectifs du projet}\label{objectifs-du-projet}

L'objectif principal du projet est de développer une plateforme capable
d'assister la génération de plans de test techniques à partir de
documents d'entrée.

Les objectifs fonctionnels sont les suivants :

\begin{itemize}
\tightlist
\item
  créer et gérer des projets ;
\item
  importer des documents techniques ;
\item
  analyser les documents importés ;
\item
  extraire automatiquement des exigences ;
\item
  générer des plans de test structurés ;
\item
  générer des cas de test détaillés ;
\item
  relier les tests aux exigences sources ;
\item
  fournir une matrice de couverture ;
\item
  permettre un échange avec un chatbot conscient du contexte du projet ;
\item
  offrir un mode interactif de validation des étapes IA ;
\item
  suivre les ressources et le planning de test ;
\item
  fournir un socle extensible pour de futures améliorations produit.
\end{itemize}

\chapter{Problématique métier}\label{probluxe9matique-muxe9tier}

Dans de nombreux projets industriels ou logiciels, la préparation des
plans de test est une tâche longue, manuelle et fortement dépendante de
l'expertise humaine.

Les principales difficultés sont :

\begin{itemize}
\tightlist
\item
  les exigences sont souvent dispersées dans plusieurs documents ;
\item
  les documents techniques peuvent être longs et complexes ;
\item
  la création de cas de test est répétitive ;
\item
  la traçabilité entre exigences et tests est difficile à maintenir ;
\item
  les équipes doivent justifier la couverture des exigences ;
\item
  les connaissances projet sont rarement capitalisées correctement ;
\item
  les outils génériques d'IA ne conservent pas une structure projet
  fiable.
\end{itemize}

Le projet propose donc une solution orientée métier : utiliser l'IA pour
accélérer la génération, tout en conservant une structure contrôlable,
traçable et réutilisable.

\section{Pourquoi une approche multi-agents
?}\label{pourquoi-une-approche-multi-agents}

Une approche multi-agents présente plusieurs avantages :

\begin{itemize}
\tightlist
\item
  meilleure séparation des responsabilités ;
\item
  prompts plus ciblés ;
\item
  débogage plus simple ;
\item
  possibilité de réviser chaque étape ;
\item
  meilleure lisibilité de la chaîne de génération ;
\item
  possibilité d'ajouter ou remplacer un agent sans réécrire tout le
  système.
\end{itemize}

\item

\item
  un type ;
\item
  des pages de début et de fin ;
\item
  un nombre de tokens.
\end{itemize}

\chapter{Limites actuelles}\label{limites-actuelles}

Le projet est fonctionnel pour une démonstration et un prototype avancé,
mais certaines limites existent :

\begin{itemize}
\tightlist
\item
  SQLite est adapté au local, mais pas idéal pour une production
  multi-utilisateur ;
\item
  les longues ingestions de documents peuvent nécessiter une meilleure
  expérience utilisateur ;
\item
  certaines fonctionnalités sont encore en évolution ;
\item
  le listing des anciennes conversations dépend encore en partie du
  navigateur côté frontend ;
\item
  les coûts LLM dépendent d'une table de prix locale qu'il faut
  maintenir ;
\item
  les résultats générés par l'IA doivent toujours être relus par un
  humain ;
\item
  le système doit être davantage testé avec des documents industriels
  réels.
\end{itemize}

\chapter{Conclusion}\label{conclusion}

Le projet \textbf{AI Test Plan Generator} démontre comment
l'intelligence artificielle peut être intégrée dans un processus métier
réel de validation et de test.

Le système permet de passer de documents techniques à des exigences,
puis à des cas de test et des plans de test traçables.

Le projet met en évidence plusieurs apprentissages importants :

\begin{itemize}
\tightlist
\item
  une application IA ne se limite pas à un prompt ;
\item
  la valeur vient de l'intégration entre interface, backend, données et
  IA ;
\item
  la traçabilité est essentielle pour un usage professionnel ;
\item
  l'humain doit garder un rôle de validation ;
\item
  les artefacts générés doivent être persistés, réutilisables et
  contrôlables.
\end{itemize}

Ce projet constitue ainsi une base solide pour un futur assistant
professionnel de génération et de gestion de plans de test.
