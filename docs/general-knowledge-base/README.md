# General Knowledge Base Test Corpus

This folder contains reusable reference documents intended for the application's **general knowledge base**, not for a single project.

The documents are realistic internal guidance documents synthesized from public standards-style references and common engineering practice.

## Documents

1. `01_Test_Process_and_Test_Plan_Guide`
2. `02_Requirements_Writing_and_Review_Guide`
3. `03_Cybersecurity_Testing_Checklist`
4. `04_Industrial_IoT_Testing_Patterns`
5. `05_Traceability_Matrix_and_Coverage_Guide`

## How To Use

Upload these documents through the app's general knowledge-base upload flow, not inside a specific project.

Then ask the chatbot or generation flow questions such as:

- What should a good test plan contain?
- How should requirements be reviewed before test generation?
- Which security tests should be considered for diagnostic bundle exports?
- How should local buffering be tested in an industrial IoT gateway?
- What is the difference between covered, partially covered, and uncovered requirements?
- How should an AI-generated plan preserve traceability?

## Difference From Project Documents

General KB documents provide reusable methodology and domain knowledge.

Project documents provide concrete requirements, architecture, constraints, and incidents for one specific product.
