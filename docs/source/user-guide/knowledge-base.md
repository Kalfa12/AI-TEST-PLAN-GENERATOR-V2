# Knowledge Base

The knowledge base is used to give the AI reusable testing context.

## General Knowledge

General knowledge should contain methodology and standards-like guidance.

Examples:

- how to structure a test plan;
- how to write good requirements;
- cybersecurity testing checklists;
- industrial IoT testing patterns;
- traceability and coverage guidance.

General knowledge should not contain one project's confidential requirement set unless it is intentionally reusable.

## Project Knowledge

Project knowledge comes from uploaded project documents and generated artifacts.

Examples:

- product requirements;
- architecture notes;
- API specifications;
- incidents;
- generated test plan;
- coverage matrix.

## Best Practice

Use the general knowledge base to teach the system how to think.

Use project documents to tell the system what to test.

For a first project, upload methodology documents into the knowledge base first, then upload project-specific PDFs into the project workspace.
