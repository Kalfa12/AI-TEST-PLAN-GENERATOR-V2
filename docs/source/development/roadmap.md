# Roadmap

The project roadmap should focus on making the prototype more reliable, more explainable, and easier to demo.

## High Priority

- Improve database concurrency beyond local SQLite limitations.
- Make all generated artifacts visible in the frontend.
- Make chat sessions easy to create, reopen, and compare.
- Show explicit chat context: project, documents, requirements, latest plan.
- Improve upload progress and background ingestion status.
- Add stronger failure recovery for long-running generation.

## Medium Priority

- Add richer dashboard metrics.
- Improve coverage gap explanation.
- Add better PDF export templates.
- Add plan versioning.
- Add requirement editing and approval status.
- Add project role management UX.

## Longer-Term

- Add production database support.
- Add persistent interactive pipeline checkpointing.
- Add organization/workspace separation.
- Add evaluation metrics for generated test quality.
- Add provider budget management with real pricing configuration.
- Add formal traceability reports for audits.
